## What to build

Das CSS für den Re-Entry-Abschnitt im HTML-Report implementieren. Das Design soll nahtlos zum bestehenden Report-Stil passen: gleiche Schriftart (Segoe UI), gleiches Blau-Farbschema (#2a5298, #1e3c72), gleiche Karten-Optik (border-radius 15px, box-shadow). Kein Tailwind-CDN, kein externes Framework — reines CSS eingebettet im `<style>`-Block des Reports.

## Acceptance criteria

- [ ] Re-Entry-Abschnitt verwendet dieselbe Schriftart wie der Rest des Reports
- [ ] Tab-Navigation ist visuell klar (aktiver Tab hervorgehoben)
- [ ] Eingabefelder, Dropdowns und Buttons sind konsistent gestylt
- [ ] Ergebnis-Bereich (blauer Kasten mit Break-Even-Kurs) ist optisch prominent
- [ ] Demo-Toggle-Button ist als fixierter Button oben rechts gestylt
- [ ] Layout ist auf Mobilgeräten (< 600px) einspaltiger (responsive)
- [ ] Kein zusätzliches CDN oder externes Stylesheet wird geladen
